# RFC — Request for Comments

## Propósito

RFCs são **propostas formais** para mudanças significativas no projeto. Permitem discussão e aprovação **antes** da implementação, preservando contexto e alternativas.

## Quando usar

- Novas features (ex: Dashboard, MCP, Marketplace)
- Mudanças arquiteturais (ex: novo protocolo, nova cloud)
- Alterações de contrato ou API
- Propostas que impactam múltiplos agentes ou módulos

## Formato

Cada RFC segue o template em [0000-template.md](0000-template.md).

## Convenções

- **Status**: Rascunho | Em discussão | Aprovado | Rejeitado | Implementado
- **Data**: YYYY-MM-DD
- **Contexto**: Por que a proposta é necessária
- **Proposta**: O que se propõe fazer
- **Alternativas**: Outras opções consideradas
- **Impacto**: Quem e o que é afetado
